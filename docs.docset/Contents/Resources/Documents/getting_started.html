Про формы

Думаю, вы умеете делать простые формы, но хотелось бы отметить пару моментов.

  • Важно указать атрибут name для полей, лучше всего будет указывать название соответствующего поля из таблицы бд.
  • для тэга form нужно обязательно указать атрибут method со значением "post"
  • для тэга form нужно указать атрибут action, где в качестве значения используется путь до файла для обработки этой формы
  • будет отлично добавить простую валидацию при помощи атрибута required для полей, обязательных к заполнению подробнее
  • можно передавать в запрос свое значение, которое пользователь не должен изменить. Для этого используется <input type="hidden"/>

form.php

<form method="post" action="add_record.php">
	<input name="field1" required />
	<textarea name="field2" required></textarea>
	<input name="field3" type="hidden" value=<?= $variable ?>/>
	<button>Создать</button>
</form>

При отправке формы пользователем, отправляется POST запрос, обрабатываемый файлом, указанным в атрибуте action. Для доступа к полям, хранящимся в теле запроса в php используется ассоциативный массив $_POST.

Обратите внимание, что имена ключей ассоциативного массива $_POST соответствуют атрибутам name у элементов формы.

Работа с бд

Хорошая статья для того чтобы дополнительно разобраться с pdo

Во-первых нужно создать базу данных в phpmyadmin. Выберите имя для бд и кодировку utf8mb4_general_ci. Также нужно будет импортировать дамп бд в панели импорт.

Подключение к бд

Создайте файл connection.php, который нужно будет импортировать при помощи require туда, где будет нужно подключение к базе данных.

connection.php

<?php
$host = "localhost";
$db = "dbname"; // название базы данных
$username = "root"; // пользователь бд
$password = ""; // пароль пользователя
  
$pdo = new  PDO("mysql:host=$host;dbname=$db", $username, $password);
$pdo->setAttribute(PDO::ATTR_ERRMODE, PDO::ERRMODE_EXCEPTION);

В этом коде объявляется переменная $pdo в которой хранится “экземпляр PDO” (подключение к бд), которая будет активно использоваться дальше. Также устанавливается атрибут ATTR_ERRMODE для того чтобы выбрасывались ошибки, которые можно обработать при помощи конструкции try catch.

Запросы к бд

Тут есть два варианта: простой запрос и подготовленный запрос с последующей передачей параметров. Второй вариант нужно использовать только когда необходимо передать в запрос любую переменную, в остальных случаях достаточно первого способа. Обратите внимание, что в обоих случаях нужно указать название таблицы вместо table

Простой запрос

<?php
require  './connection.php';
$stmt = $pdo->query('SELECT  *  FROM table');

Здесь импортируется файл connection.php для того чтобы иметь доступ к переменной $pdo, объявленной в нем. Выполняется SELECT sql запрос при помощи метода query объекта $pdo.

Выполнение подготовленного запроса с последующей передачей параметров

<?php
require  './connection.php';
$stmt = $pdo->prepare('SELECT  *  FROM table WHERE id = ?');
$stmt->execute([$id]);

Здесь импортируется файл connection.php для того чтобы иметь доступ к переменной $pdo, объявленной в нем. Выполняется метод prepare, который создает подготовленный запрос, тут в sql нужно указать символ ? для того чтобы потом подставить туда переменную. Вызвав метод execute у подготовленного запроса, можно передать параметры в виде массива

Вывод данных из бд

<?php
require  './connection.php';
$stmt = $pdo->query('SELECT  *  FROM table');
?>

<ul>
<?php foreach ($stmt as $row) { ?>
	<li><b><?= $row['property1'] ?></b>: <?= $row['property2'] ?></li>
<?php } ?>
</ul>

Здесь выполняется простой запрос. Данные выводятся в виде списка при помощи конструкции foreach, обратите внимание, что переменная $row это ассоциативный массив, где ключами являются названия полей в таблице.

Добавление записи в бд

Для начала нужно создать простую html форму.

form.html

<form method="post" action="add_record.php">
	<input name="field1" required />
	<textarea name="field2" required></textarea>
	<button>Создать</button>
</form>

Теперь нужно создать файл для обработки формы. После импорта файла с подключением, для удобства можно создать отдельные переменные для ключей $_POST. Остается только выполнить подготовленный запрос.

add_record.php

<?php
require './connection.php'; // импортировать файл с подключением

$field1 = $_POST['field1'];
$field2 = $_POST['field1'];

$stmt->prepare("INSERT INTO table (field1, field1) values (?, ?)");
$stmt->execute([$title, $content]);

Редактирование записи

Все точно также как и при создании записи, меняется только sql запрос. Тут можно использовать скрытое поле для передачи поля для идентификации. Также стоит указывать значение по умолчанию для полей при помощи атрибута value

form.php

<form method="post" action="edit_record.php">
	<input type="hidden" name="field1" value="<?= $record['field1'] ?>" />
	<input name="field2" value="<?= $record['field2'] ?>" required />
	<input name="field3" value="<?= $record['field3'] ?>" required />
	<button>Изменить</button>
</form>

edit_record.php

<?php
require './connection.php'; // импортировать файл с подключением

$field1 = $_POST['field1'];
$field2 = $_POST['field2'];
$field3 = $_POST['field3 '];

$stmt->prepare("UPDATE table SET field1 = ?, field2 = ? WHERE field3 = ?");
$stmt->execute([$field1, $field2, $field3]);

Удаление записи

Все точно также как и при создании записи, меняется только sql запрос. Тут можно использовать скрытое поле для передачи поля для идентификации.

post.php

<form method="post" action="delete_record.php">
	<input type="hidden" name="field1" value="<?= $record['field1'] ?>" />
	<button>Удалить</button>
</form>

delete_record.php

<?php
require './connection.php'; // импортировать файл с подключением

$field1 = $_POST['field1'];

$stmt->prepare("DELETE FROM posts WHERE field1 = ?");
$stmt->execute([$field1]);
