Про формы

Думаю, вы умеете делать простые формы, но хотелось бы отметить пару моментов.

  • Важно указать атрибут name для полей, лучше всего будет указывать название соответствующего поля из таблицы бд.
  • для тэга form нужно обязательно указать атрибут method со значением "post"
  • для тэга form нужно указать атрибут action, где в качестве значения используется путь до файла для обработки этой формы
  • будет отлично добавить простую валидацию при помощи атрибута required для полей, обязательных к заполнению подробнее
  • можно передавать в запрос свое значение, которое пользователь не должен изменить. Для этого используется <input type="hidden"/>

form.php

<form method="post" action="add_record.php">
	<input name="field1" required />
	<textarea name="field2" required></textarea>
	<input name="field3" type="hidden" value=<?= $variable ?>/>
	<button>Создать</button>
</form>

При отправке формы пользователем, отправляется POST запрос, обрабатываемый файлом, указанным в атрибуте action. Для доступа к полям, хранящимся в теле запроса в php используется ассоциативный массив $_POST.

Обратите внимание, что имена ключей ассоциативного массива $_POST соответствуют атрибутам name у элементов формы.

Работа с бд

Хорошая статья для того чтобы дополнительно разобраться с pdo

Во-первых нужно создать базу данных в phpmyadmin. Выберите имя для бд и кодировку utf8mb4_general_ci. Также нужно будет импортировать дамп бд в панели импорт.

Подключение к бд

Создайте файл connection.php, который нужно будет импортировать при помощи require туда, где будет нужно подключение к базе данных.

connection.php

<?php
$host = "localhost";
$db = "dbname"; // название базы данных
$username = "root"; // пользователь бд
$password = ""; // пароль пользователя
  
$pdo = new  PDO("mysql:host=$host;dbname=$db", $username, $password);
$pdo->setAttribute(PDO::ATTR_ERRMODE, PDO::ERRMODE_EXCEPTION);

В этом коде объявляется переменная $pdo в которой хранится “экземпляр PDO” (подключение к бд), которая будет активно использоваться дальше. Также устанавливается атрибут ATTR_ERRMODE для того чтобы выбрасывались ошибки, которые можно обработать при помощи конструкции try catch.

Запросы к бд

Тут есть два варианта: простой запрос и подготовленный запрос с последующей передачей параметров. Второй вариант нужно использовать только когда необходимо передать в запрос любую переменную, в остальных случаях достаточно первого способа. Обратите внимание, что в обоих случаях нужно указать название таблицы вместо table

Простой запрос

<?php
require  './connection.php';
$stmt = $pdo->query('SELECT  *  FROM table');

Здесь импортируется файл connection.php для того чтобы иметь доступ к переменной $pdo, объявленной в нем. Выполняется SELECT sql запрос при помощи метода query объекта $pdo.

Выполнение подготовленного запроса с последующей передачей параметров

<?php
require  './connection.php';
$stmt = $pdo->prepare('SELECT  *  FROM table WHERE id = ?');
$stmt->execute([$id]);

Здесь импортируется файл connection.php для того чтобы иметь доступ к переменной $pdo, объявленной в нем. Выполняется метод prepare, который создает подготовленный запрос, тут в sql нужно указать символ ? для того чтобы потом подставить туда переменную. Вызвав метод execute у подготовленного запроса, можно передать параметры в виде массива

Вывод данных из бд

<?php
require  './connection.php';
$stmt = $pdo->query('SELECT  *  FROM table');
?>

<ul>
<?php foreach ($stmt as $row) { ?>
	<li><b><?= $row['property1'] ?></b>: <?= $row['property2'] ?></li>
<?php } ?>
</ul>

Здесь выполняется простой запрос. Данные выводятся в виде списка при помощи конструкции foreach, обратите внимание, что переменная $row это ассоциативный массив, где ключами являются названия полей в таблице.

Редактирование записи

Все точно также как и при создании записи, меняется только sql запрос. Тут можно использовать скрытое поле для передачи поля для идентификации. Также стоит указывать значение по умолчанию для полей при помощи атрибута value

form.php

<form method="post" action="edit_record.php">
	<input type="hidden" name="field1" value="<?= $record['field1'] ?>" />
	<input name="field2" value="<?= $record['field2'] ?>" required />
	<input name="field3" value="<?= $record['field3'] ?>" required />
	<button>Изменить</button>
</form>

edit_record.php

<?php
require './connection.php'; // импортировать файл с подключением

$field1 = $_POST['field1'];
$field2 = $_POST['field2'];
$field3 = $_POST['field3 '];

$stmt->prepare("UPDATE table SET field1 = ?, field2 = ? WHERE field3 = ?");
$stmt->execute([$field1, $field2, $field3]);

Авторизация

хорошая статься про авторизацию на php, все что написано дальше было бы взято из нее. Так же там приведен простой способ обработки ошибок при помощи функции flash()

Самый просто способ сделать авторизацию - это хранить id пользователя (или все данные о пользователе) в сессии. Для этого в php используется $_SESSION.

Предварительно нужно инициализировать сессию при помощи session_start(). Эту функцию можно вызвать в файле, который везде подключается, например в connection.php (дальше он называется boot.php).

boot.php

<?php
$host = "localhost";
$db = "dbname"; // название базы данных
$username = "root"; // пользователь бд
$password = ""; // пароль пользователя
  
$pdo = new  PDO("mysql:host=$host;dbname=$db", $username, $password);
$pdo->setAttribute(PDO::ATTR_ERRMODE, PDO::ERRMODE_EXCEPTION);

session_start()

Регистрация

registration.html

<form method="POST" action="register.php">
	<input name="login" placeholder="Логин" required />
	<input name="password" placeholder="Пароль" type="password" required />
	<button>Зарегистрироваться</button>
</form>

После импорта файла boot.php, для удобства можно создать отдельные переменные для ключей $_POST.

Обратите внимание что пароль хэшируется при помощи функции password_hash()

Дальше выполняется подготовленный запрос. Если все получилось, то пользователь перенаправляется на страницу входа, иначе ему показывается ошибка.

register.php

<?php
require  './boot.php';

$login = $_POST['login'];
$password = password_hash($_POST['password'], PASSWORD_DEFAULT);

try {
	$stmt = $pdo->prepare('INSERT INTO users (login, password) values (?, ?)');
	$success = $stmt->execute([$login, $password]);

	if ($success) {
		header('Location: /login.php');
		exit;
	}
} catch (PDOException  $e) {
	echo $e->getMessage();
}

Вход

login.html

<form method="POST" action="do_login.php">
	<input name="login" placeholder="Логин"  required />
	<input name="password" placeholder="Пароль"  type="password" required />
	<button>Войти</button>
</form>

do_login.php

<?php
require  './boot.php';

$login = $_POST['login'];
$password = $_POST['password'];

try {
	$stmt = $pdo->prepare('SELECT  *  FROM users WHERE  login  = ?');
	$stmt->execute([$login]);
	$user = $stmt->fetch(PDO::FETCH_ASSOC);
	if (!$user) {
		throw  new  Exception('Пользователя с таким логином не существует');
	} else {
		if (!password_verify($password, $user['password'])) {
			throw  new  Exception('Неверный пароль');
		} else {
			$_SESSION['user'] = $user;
			header('Location: /');
			exit;
		}
	}
} catch (Exception  $e) {
	echo $e->getMessage();
}

После импорта файла boot.php, для удобства можно создать отдельные переменные для ключей $_POST.

Дальше нужно получить пользователя по указанному логину, если такого пользователя нет, нужно бросить ошибку.

Если пользователь существует, тогда нужно сверить пароли, делается это при помощи функции password_verify(). Если пароль неверный, тогда тоже выбрасывается ошибка, иначе пользователь добавляется в сессию и теперь считается авторизованным.

В конце происходит перенаправление на главную страницу.

Выход

Для выхода достаточно обнулить значение $_SESSION['user'].

logout.php

<?php
require  './boot.php';
unset($_SESSION['user_id']);
header('Location: /');

Приведенный выше пример, можно использовать как в качестве action для формы, так и через ссылку

<form action="logout.php">
	<button>Выйти</button>
</form>

<!-- или --> 

<a href="logout.php">Выйти</a>

Получение пользователя

Для того чтобы получить пользователя достаточно обратиться к $_SESSION['user']

<?php
require  './boot.php';

$user = $_SESSION['user'];

if (!$user) {
	header('Location: /login.php');
	exit;
}
?>

<p>Добро пожаловать, <?= $user['login'] ?>!</p>

В этом примере если пользователь не авторизован, то его перенаправляет на страницу входа, иначе отображается приветственное сообщение.

Роли пользователей

Для реализации системы ролей достаточно иметь поле role в таблице пользователей, со значением по умолчанию user для регистрации обычных пользователей.

Для создания учетной записи администратора нужно зарегистрировать обычного пользователя, после чего в phpmyadmin сменить значение поля role с user на admin.

В примере ниже если роль пользователя не admin, то его перенаправляет на главную страницу.

admin.php

<?php
require  './boot.php';

$user = $_SESSION['user'];

if (!$user) {
	header('Location: /login.php');
	exit;
}

if ($user['role'] !== 'admin') {
	header('Location: /');
	exit;
}

Обработка ошибок

flash.php

function  flash(?string $message = null)
{
	if ($message) {
		$_SESSION['flash'] = $message;
	} else {
		if (!empty($_SESSION['flash'])) { ?>
			<p><?=  $_SESSION['flash'] ?></p>
		<?php }
		unset($_SESSION['flash']);
	}
}

Добавьте эту функцию в boot.php. Если нужно добавить ошибку вызовите flash('your-error-message'). Если вывести, то просто вызовите, не передавая аргументов flash() в том месте, где хотите ее отобразить. Разумеется, чтобы пользоваться этой функцией, сначала нужно импортировать boot.php.
